Go通道

2020-02-26

通道channel与goroutine，并驾齐驱。

Don't communicate by sharing memory;share momory by communicating.

通道的值本身就是并发安全，也是Go语言自带的、唯一一个可以满足并发安全的类型。可以利用通道在不同goroutine之间传递数据。

声明并初始化一个通道时，使用Go的内建函数make来初始化，就像make用来初始化一个切片一样。

var a chan int

类型字面量chan int,其中chan是标识通道类型的关键字，而int则说明通道类型的元素类型。

var a = make(chan int, 1)

初始化通道的时候，make还可以接收一个参数int,这个参数代表int的容量。

当参数为0时候，通道为无缓冲通道。当参数大于0，为缓冲通道。

这里的通道相当于一个先进先出队列（FIFO）。也就是说通道的元素是按照严格的发送顺序。先发送的一定先被接收。

发送和接收都用到操作符<-,表示了通道数据的流动方向

ch1 := make(chan int, 3)
ch1 <- 2
ch1 <- 1
ch1 <- 7
elem1 := <-ch1

通道的发送和接收都有哪些特性

  1. 对于同一个通道，发送操作之间是互斥的，接收操作之间也是互斥的
  2. 发送操作和接收操作对元素处理都是不可分割
  3. 发送操作在完成之前会被阻塞掉，接收操作在完成之前也会阻塞掉

元素从外界进入通道时会被复制，更具体的说是元素的副本。
